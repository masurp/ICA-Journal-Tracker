# ICA Journal Tracker

A lightweight, automatically updated dashboard tracking the most read, most cited, and latest papers from all six [International Communication Association (ICA)](https://www.icahdq.org/) flagship journals.

**Live site:** [masurp.github.io/ICA-Journal-Tracker](https://masurp.github.io/ICA-Journal-Tracker/)

---

## Journals covered

| Journal | Abbreviation |
|---------|-------------|
| Journal of Communication | JoC |
| Human Communication Research | HCR |
| Communication Theory | CT |
| Journal of Computer-Mediated Communication | JCMC |
| Communication, Culture & Critique | CCC |
| Annals of the International Communication Association | Annals |

## How it works

A [GitHub Actions](.github/workflows/update_data.yml) workflow runs every **Monday at 06:00 UTC** and:

1. Queries the **[Crossref API](https://www.crossref.org/)** for the most cited and most recently published papers per journal (filtered by ISSN)
2. Enriches each paper with topic labels from the **[Semantic Scholar API](https://www.semanticscholar.org/product/api)**
3. Fetches **[Altmetric Attention Scores](https://www.altmetric.com/)** as a proxy for "most read" (publisher download counts are not publicly available)
4. Writes static JSON files to `data/` and commits them back to `main`
5. GitHub Pages serves the updated site automatically

The frontend is plain HTML/CSS/JS — no build step, no framework.

---

## Data sources & notes

- **Most Cited** — ranked by total citation count from Crossref
- **Latest** — sorted by publication date (Crossref)
- **Most Read** — approximated by [Altmetric Attention Score](https://www.altmetric.com/about-our-data/the-donut-and-score/), which aggregates mentions in news outlets, blogs, social media, Wikipedia, and policy documents. Publisher download counts are proprietary and not publicly available.
- **Topic labels** — generated by the Semantic Scholar ML classifier (`s2FieldsOfStudy`)

## Local setup

```bash
git clone https://github.com/masurp/ICA-Journal-Tracker.git
cd ICA-Journal-Tracker/scripts
pip3 install -r requirements.txt
MAILTO=your@email.com python3 fetch_data.py
```

Setting `MAILTO` is optional but recommended — it routes Crossref requests through their "polite pool" with higher rate limits.

Open `index.html` in a browser to view the site locally.

---

## GitHub Actions setup

To run the weekly workflow in your own fork:

1. Go to **Settings → Secrets and variables → Actions**
2. Add a repository secret: `CROSSREF_MAILTO` = your email address
3. Enable GitHub Pages: **Settings → Pages → Source: branch `main`, folder `/`**
4. Trigger the first run manually: **Actions → Update Journal Data → Run workflow**
